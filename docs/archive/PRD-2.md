# VoiceMate 功能补充说明

## 高级配置与历史记录模块

---

### 一、本地配置文件（ASR + LLM 全量配置）

#### 1.1 概述

支持用户通过本地配置文件（JSON/YAML）管理所有引擎参数，满足极客用户对版本控制、批量修改和自动化工作流的需求。

#### 1.2 配置文件规范

**文件路径**：`~/Library/Application Support/VoiceMate/config.json`

**配置结构**：

```json
{
  "version": "1.0",
  "general": {
    "hotkey": "Cmd+Shift+V",
    "launchAtStartup": false,
    "windowStyle": "vibrancy"
  },
  "asr": {
    "engine": "system",  // system | whisper | iflytek | aliyun | openai-whisper
    "system": {
      "language": "zh-CN"
    },
    "whisper": {
      "modelPath": "/Users/username/Models/ggml-base.bin",
      "threads": 4,
      "language": "auto"
    },
    "iflytek": {
      "appId": "",
      "apiKey": ""
    },
    "aliyun": {
      "accessKey": "",
      "accessSecret": ""
    },
    "openaiWhisper": {
      "apiKey": "",
      "model": "whisper-1"
    }
  },
  "llm": {
    "enabled": false,
    "engine": "ollama",  // ollama | openai | deepseek | claude | custom
    "ollama": {
      "baseUrl": "http://localhost:11434",
      "model": "qwen2.5:7b",
      "temperature": 0.7,
      "numPredict": 512
    },
    "openai": {
      "apiKey": "",
      "model": "gpt-4o-mini",
      "baseUrl": "https://api.openai.com/v1",
      "temperature": 0.7
    },
    "deepseek": {
      "apiKey": "",
      "model": "deepseek-v4-flash",
      "baseUrl": "https://api.deepseek.com/v1",
      "temperature": 0.7
    },
    "claude": {
      "apiKey": "",
      "model": "claude-3-5-sonnet-20241022",
      "baseUrl": "https://api.anthropic.com/v1",
      "temperature": 0.7
    },
    "custom": {
      "apiKey": "",
      "model": "",
      "baseUrl": "",
      "temperature": 0.7
    },

      "system": "你是一个专业的文字润色助手，请将口语改写为书面语。",
      "user": "请将以下口语内容改写成正式书面中文：\n1. 去掉'嗯、啊、那个、就是说、其实'等口头禅\n2. 修正错别字和语法错误\n3. 保持原意不变\n4. 只输出改写后的文本，不要任何解释或前缀\n\n口语内容：{{input}}\n\n改写结果："

```

#### 1.3 交互模式

| 模式 | 描述 | 优先级 |
|------|------|--------|
| **UI 优先** | 用户在设置界面修改任何选项，自动同步写入配置文件 | P0 |
| **配置文件优先** | App 启动时读取配置文件；若文件被外部修改，自动热重载（无需重启） | P1 |
| **冲突处理** | 若配置文件格式错误，App 回退至上次有效配置并提示用户 | P1 |

#### 1.4 配置导入/导出

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 导出配置 | 一键导出当前配置为 JSON 文件（API Key 字段自动脱敏为 `"****"`） | P2 |
| 导入配置 | 选择 JSON 文件导入，覆盖当前配置（含 API Key 则写入 Keychain） | P2 |
| 重置默认 | 一键恢复出厂设置（清空所有用户自定义内容） | P1 |

### 二、自定义 Prompt 编辑器

#### 2.1 功能描述

允许高级用户完全自定义润色 Prompt，不再受限于 App 内置的固定模板。

#### 2.2 Prompt 变量支持

| 变量 | 描述 | 示例 |
|------|------|------|
| `{{input}}` | ASR 原始识别文本（必填） | 用户说话的内容 |
| `{{language}}` | 当前系统语言 | `zh-CN` |
| `{{engine}}` | 当前使用的 ASR 引擎名称 | `whisper` |
| `{{timestamp}}` | 当前时间戳 | `2026-07-05 14:30:00` |
| `{{custom.xxx}}` | 用户自定义变量（通过配置文件扩展） | 如 `{{custom.tone}}` = `正式` |

#### 2.3 Prompt 模板管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 单模板编辑 | 设置界面提供多行文本框，编辑 System Prompt 和 User Prompt | P0 |
| 多模板切换 | 用户可保存多套 Prompt 模板（如“正式公文版”、“营销文案版”、“代码注释版”）并快速切换 | P1 |
| 模板命名 | 每套模板可自定义名称 | P1 |
| 模板导入/导出 | 支持单个模板导出为 `.vmprompt` 文件，方便分享 | P2 |

#### 2.4 编辑器界面设计

```

│  ├─────────────────────────────────────────────────┤ │
│  │  System Prompt：                                │ │
│  │  ┌─────────────────────────────────────────────┐ │ │

│  │  User Prompt：                                  │ │

│  │  │ {{input}}                                   │ │ │
│  │  └─────────────────────────────────────────────┘ │ │
```

### 三、本地历史记录

#### 3.1 功能描述

自动保存用户最近的语音输入及润色结果，支持快速回溯和复用。

#### 3.2 存储规范

**存储位置**：`~/Library/Application Support/VoiceMate/history.json`（或 SQLite）

**数据结构**：

```json
{
  "history": [
    {
      "id": "uuid-xxx",
      "timestamp": "2026-07-05T14:30:00+08:00",
      "asrResult": "我想今天下午去开会然后讨论一下那个项目进度",
      "llmResult": "我计划今天下午参加会议并讨论项目进度。",
      "engine": "system",
      "llmEngine": "ollama",
      "favorite": false
    }
  ],
  "maxCount": 20
}
```

#### 3.3 交互方式

| 交互场景 | 操作方式 | 优先级 |
|----------|----------|--------|
| **悬浮窗快速翻阅** | 悬浮窗激活时，按键盘 `↑` / `↓` 键翻阅历史记录，按 `Enter` 直接粘贴选中的条目 | P1 |
| **历史面板** | 点击菜单栏图标 → “历史记录”，打开完整历史列表 | P1 |
| **右键菜单** | 在历史列表中右键 → “复制原文”、“复制润色结果”、“删除” | P1 |

#### 3.4 历史记录列表界面

```

│  │    14:25  帮我写个邮件给客户                           │ │
```

#### 3.5 隐私控制

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 退出清空 | 开关：App 退出时自动清空所有历史记录 | P2 |
| 禁用记录 | 开关：完全禁用历史记录功能（数据不落盘） | P2 |
| 手动清空 | 历史面板提供“清空全部”按钮 | P1 |
| 单条删除 | 每条历史记录支持单独删除 | P1 |
| 收藏标记 | 重要记录可收藏（⭐），防止被自动清理 | P2 |

### 四、补充后的设置界面变更

#### 新增 Tab：高级设置

| 设置项 | 类型 | 默认值 | 优先级 |
|--------|------|--------|--------|
| **配置文件路径** | 只读显示 | `~/Library/Application Support/VoiceMate/config.json` | P1 |
| [导出配置] | 按钮 | - | P2 |
| [导入配置] | 按钮 | - | P2 |
| [重置所有配置] | 按钮（危险操作，需二次确认） | - | P1 |
| **历史记录数量** | 数字选择器（5–100） | 20 | P1 |
| [清空历史记录] | 按钮 | - | P1 |
| 退出时自动清空历史 | 开关 | 关闭 | P2 |
| 禁用历史记录 | 开关 | 关闭 | P2 |

#### Prompt 编辑器：独立页面或弹窗

从设置界面“LLM 设置”区域点击 [编辑 Prompt] 按钮进入。

### 五、技术实现补充

| 层级 | 技术选型 |
|------|----------|
| 配置文件读写 | `JSONEncoder` / `JSONDecoder` + `FileManager`（监控文件变更用 `DispatchSource.FileSystemEvent`） |
| 热重载 | 监听配置文件修改事件，解析成功后动态切换引擎参数 |
| 历史存储 | SQLite（`GRDB.swift`）或 JSON 文件（根据数据量决定，20条用 JSON 即可） |
| 配置与 Keychain 协同 | API Key 等敏感字段存储在 Keychain，配置文件中仅存储占位符或引用 ID |

---

### 六、设计原则补充

6. **配置即代码**：一切配置可文件化，方便版本管理和分享
7. **历史即安全**：让用户敢于表达，因为每句话都有迹可循
8. **Prompt 即权力**：用户对输出风格的完全控制，是 VoiceMate 的核心竞争力
